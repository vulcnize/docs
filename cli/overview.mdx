## Usage

```bash
vulcn [command] [options]
```

## Global Options

| Option          | Description               |
| --------------- | ------------------------- |
| `-V, --version` | Output the version number |
| `-h, --help`    | Display help for command  |

## Quick Reference

| Command                     | Description                    |
| --------------------------- | ------------------------------ |
| `vulcn init [url]`          | Create `.vulcn.yml` project    |
| `vulcn record [url]`        | Record browser interactions    |
| `vulcn crawl [url]`         | Auto-discover forms and inputs |
| `vulcn run`                 | Run security tests             |
| `vulcn store [user] [pass]` | Store encrypted credentials    |
| `vulcn payloads`            | List available payload types   |
| `vulcn install`             | Install browsers               |
| `vulcn doctor`              | Check environment              |

<Note>
  Most commands read configuration from `.vulcn.yml`. CLI flags override config
  values for that run only.
</Note>

## Examples

### Basic Workflow

```bash
# Initialize project
vulcn init https://example.com

# Auto-discover forms
vulcn crawl

# Run with default payloads from .vulcn.yml
vulcn run

# Override payload types for this run
vulcn run -p xss sqli
```

### Browser Management

```bash
# Check which browsers are installed
vulcn doctor

# Install Chromium (default)
vulcn install

# Install all browsers
vulcn install --all
```

### Authenticated Scanning

```bash
# Store credentials (encrypted to auth/state.enc)
vulcn store admin password

# Crawl authenticated pages (auth auto-discovered)
vulcn crawl

# Run authenticated scans (auth auto-discovered)
vulcn run -p xss sqli -r html
```

### Project Structure

All commands work with the convention-based project structure:

```
my-project/
├── .vulcn.yml          ← config (single source of truth)
├── sessions/           ← recorded/crawled sessions
├── auth/               ← encrypted credentials
└── reports/            ← generated reports
```
