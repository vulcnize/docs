## Usage

```bash
vulcn init [url] [options]
```

## Arguments

| Argument | Description                      | Required |
| -------- | -------------------------------- | -------- |
| `url`    | Target URL to pre-fill in config | No       |

## Options

| Option        | Description                    |
| ------------- | ------------------------------ |
| `-f, --force` | Overwrite existing config file |

## Description

The `init` command creates a `.vulcn.yml` file in the current directory, marking it as a Vulcn project. It also creates the `sessions/` directory for storing recorded and crawled sessions.

## Generated Configuration

The generated `.vulcn.yml`:

```yaml
# Vulcn Project Configuration
# Docs: https://docs.vulcn.dev/config
#
# This file is the single source of truth for your Vulcn project.
# All commands auto-discover this file from the current directory.
#
# Convention:
#   sessions/   ‚Üê recorded/crawled sessions
#   auth/       ‚Üê encrypted credentials
#   reports/    ‚Üê generated reports

target: https://example.com

scan:
  browser: chromium
  headless: true
  timeout: 30000

payloads:
  types:
    - xss
  payloadbox: false
  limit: 100
  custom: null

detection:
  xss:
    dialogs: true
    console: true
    domMutation: false
    severity: high
  reflection:
    enabled: true
  passive: true

crawl:
  depth: 2
  maxPages: 20
  sameOrigin: true
  timeout: 10000

report:
  format: html
```

## Configuration Sections

### target

The URL to scan. Used by `vulcn crawl` and `vulcn record` as the default target.

### scan

Browser settings for scanning and recording:

- **browser: chromium** ‚Äî Browser engine (chromium, firefox, webkit)
- **headless: true** ‚Äî Run tests without visible window
- **timeout: 30000** ‚Äî Per-step timeout in milliseconds

### payloads

Payload configuration:

- **types** ‚Äî Payload types to use (xss, sqli, xxe, cmd, redirect, traversal)
- **payloadbox** ‚Äî Opt-in to PayloadsAllTheThings community payloads
- **limit** ‚Äî Max payloads per type
- **custom** ‚Äî Path to custom payload YAML file

### detection

Detection plugin settings:

- **xss** ‚Äî XSS detection via dialogs, console markers, DOM mutation
- **reflection** ‚Äî Input reflection detection
- **passive** ‚Äî Enable passive security checks (headers, cookies, CORS)

### crawl

Crawler settings:

- **depth** ‚Äî Max link-follow depth
- **maxPages** ‚Äî Max pages to visit
- **sameOrigin** ‚Äî Stay on same origin

### report

Report generation:

- **format** ‚Äî Output format (html, json, yaml, sarif, all)
