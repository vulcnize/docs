## Overview

The report plugin generates branded, professional security reports at the end of each `vulcn run`. It supports three output formats and integrates seamlessly with CI/CD pipelines.

## Installation

The report plugin is bundled with the `vulcn` CLI. No additional installation needed.

For programmatic use:

<CodeGroup>

```bash npm
npm install @vulcn/plugin-report
```

```bash pnpm
pnpm add @vulcn/plugin-report
```

</CodeGroup>

## CLI Usage

### Generate an HTML Report

```bash
vulcn run session.vulcn.yml --report html
```

The HTML report is saved to the current directory and automatically opens in your browser.

### Generate JSON for CI/CD

```bash
vulcn run session.vulcn.yml --report json
```

### Generate All Formats

```bash
vulcn run session.vulcn.yml --report all
```

### Custom Output Directory

```bash
vulcn run session.vulcn.yml --report html --report-output ./reports
```

## CLI Options

| Option                  | Description                                  | Default |
| ----------------------- | -------------------------------------------- | ------- |
| `-r, --report <format>` | Report format: `html`, `json`, `yaml`, `all` | -       |
| `--report-output <dir>` | Output directory for generated reports       | `.`     |

## Configuration

### Via `vulcn.config.yml`

```yaml
plugins:
  - name: "@vulcn/plugin-report"
    config:
      format: html
      outputDir: ./reports
```

### Config Options

| Option      | Type     | Default | Description                                  |
| ----------- | -------- | ------- | -------------------------------------------- |
| `format`    | `string` | `html`  | Output format: `html`, `json`, `yaml`, `all` |
| `outputDir` | `string` | `.`     | Directory to write report files              |

## CI/CD Integration

### GitHub Actions

```yaml
- name: Run security tests
  run: vulcn run session.vulcn.yml --report json --report-output ./reports

- name: Upload report
  uses: actions/upload-artifact@v4
  with:
    name: security-report
    path: ./reports/
```

### GitLab CI

```yaml
security_scan:
  script:
    - vulcn run session.vulcn.yml --report json --report-output ./reports
  artifacts:
    paths:
      - reports/
    reports:
      security: reports/vulcn-report.json
```

### Exit Codes

| Code | Meaning                  |
| ---- | ------------------------ |
| `0`  | No vulnerabilities found |
| `1`  | Vulnerabilities detected |

Use the exit code to gate deployments:

```bash
vulcn run session.vulcn.yml --report all || exit 1
```

## Programmatic Usage

```typescript
import { generateHtml, generateJson, generateYaml } from "@vulcn/plugin-report";

const reportData = {
  session,
  result,
  generatedAt: new Date().toISOString(),
  engineVersion: "0.3.0",
};

// Generate HTML string
const html = generateHtml(reportData);

// Generate JSON string
const json = generateJson(reportData);

// Generate YAML string
const yaml = generateYaml(reportData);
```
