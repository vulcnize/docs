## Prerequisites

<Info>**Requirements:** - Node.js v18 or higher - npm, pnpm, or yarn</Info>

## Installation

<Steps>
<Step title="Install the CLI">

<CodeGroup>

```bash npm
npm install -g vulcn
```

```bash pnpm
pnpm add -g vulcn
```

```bash yarn
yarn global add vulcn
```

</CodeGroup>

</Step>

<Step title="Initialize a project">

Create a `.vulcn.yml` project config:

```bash
vulcn init https://your-target.com
```

This creates a config file and `sessions/` directory:

```yaml
# .vulcn.yml
target: https://your-target.com

scan:
  browser: chromium
  headless: true
  timeout: 30000

payloads:
  types:
    - xss
  payloadbox: false
  limit: 100

detection:
  xss:
    dialogs: true
    console: true
    domMutation: false
    severity: high
  reflection:
    enabled: true
  passive: true

crawl:
  depth: 2
  maxPages: 20
  sameOrigin: true

report:
  format: html
```

</Step>

<Step title="Install browsers (optional)">

Vulcn uses Playwright for browser automation. Browsers are installed automatically on first use, but you can install them manually:

```bash
vulcn install
```

Or install all browsers:

```bash
vulcn install --all
```

</Step>
</Steps>

## Record Your First Session

<Steps>
<Step title="Start recording">

Open a browser and record your interactions with the target application:

```bash
vulcn record
```

The target URL is read from `.vulcn.yml`. You can also override it:

```bash
vulcn record https://example.com
```

<Tip>
  The browser opens and records everything you do. Fill out forms, click
  buttons, navigate pagesâ€”Vulcn captures it all.
</Tip>

</Step>

<Step title="Interact with the app">

In the browser:

1. Navigate to a page with input fields
2. Fill out a form (login, search, comment, etc.)
3. Submit the form
4. Press **Ctrl+C** to stop recording

</Step>

<Step title="Session saved">

Your interactions are saved to `sessions/`:

```yaml
# sessions/recording-2026-02-14.yml
name: Session 2026-02-14
driver: browser
driverConfig:
  startUrl: https://example.com
  browser: chromium
steps:
  - id: step_001
    type: browser.navigate
    url: https://example.com
  - id: step_002
    type: browser.input
    selector: input[name="search"]
    value: test query
    injectable: true
  - id: step_003
    type: browser.click
    selector: button[type="submit"]
```

</Step>
</Steps>

## Or, Auto-Crawl Instead

Skip manual recording entirely â€” let Vulcn discover forms automatically:

```bash
vulcn crawl
```

This finds all forms and injectable inputs across the site and generates session files in `sessions/`. The target URL is read from `.vulcn.yml`. See [vulcn crawl](/cli/crawl) for details.

## Run Security Tests

<Steps>
<Step title="Execute tests">

Run all sessions with security payloads:

```bash
vulcn run
```

Vulcn reads everything from `.vulcn.yml` â€” payload types, detection settings, browser config â€” and loads all sessions from `sessions/`.

</Step>

<Step title="Override payload types">

Test with specific payload types (overrides `.vulcn.yml`):

```bash
vulcn run -p xss sqli
```

Available types: `xss`, `sqli`, `xxe`, `cmd`, `redirect`, `traversal`

</Step>

<Step title="Review findings">

<Step title="Generate a report">

Generate a professional HTML security report:

```bash
vulcn run -r html
```

This creates a `vulcn-report.html` in `reports/` and opens it in your browser. The report includes a risk gauge, severity breakdown, and detailed finding cards with payloads and evidence.

For CI/CD pipelines, use JSON or SARIF:

```bash
vulcn run -r json
vulcn run -r sarif
```

</Step>
</Steps>

## Authenticated Scanning

For applications behind login pages, Vulcn can perform authenticated crawling and scanning.

<Steps>
<Step title="Store credentials">

```bash
vulcn store admin password
```

Credentials are encrypted with AES-256-GCM and saved to `auth/state.enc`. Set `VULCN_KEY` env var or enter a passphrase when prompted.

<Tip>
  If your `.vulcn.yml` has an `auth` section with `loginUrl`, Vulcn uses it
  automatically. Otherwise, pass `--login-url` on the CLI.
</Tip>

</Step>

<Step title="Crawl with authentication">

```bash
vulcn crawl
```

Vulcn auto-discovers `auth/state.enc`, decrypts credentials, performs the login, captures cookies/storage, and crawls all authenticated pages.

</Step>

<Step title="Scan with authentication">

```bash
vulcn run -p xss sqli -r html
```

The scanner inherits the authenticated browser state, so it can test forms behind login. Auth is auto-discovered â€” no `--creds` flag needed.

</Step>
</Steps>
