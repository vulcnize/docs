## Overview

The payloads plugin handles all payload loading for Vulcn:

- **PayloadsAllTheThings** — community-curated payloads fetched on demand (default)
- **Custom files** — expert-provided YAML/JSON payload files

Payloads are sourced from [PayloadsAllTheThings](https://github.com/swisskyrepo/PayloadsAllTheThings), the largest collection of security payloads and vulnerability testing techniques.

## Installation

The plugin is included with the CLI and loaded automatically.

```bash
npm install @vulcn/plugin-payloads
```

## Configuration

```yaml
plugins:
  - name: "@vulcn/plugin-payloads"
    config:
      types:
        - xss
        - sqli
      limit: 50
      files:
        - ./custom-payloads.yml
```

## Options

| Option  | Type     | Default | Description                                   |
| ------- | -------- | ------- | --------------------------------------------- |
| `types` | string[] | -       | Payload types to load (accepts short aliases) |
| `limit` | number   | `50`    | Maximum payloads per type                     |
| `files` | string[] | -       | Custom payload file paths (YAML/JSON)         |

## Payload Types

| Type                | Short Aliases      | Description                                     |
| ------------------- | ------------------ | ----------------------------------------------- |
| `xss`               | —                  | Cross-Site Scripting — script injection, events |
| `sql-injection`     | `sqli`, `sql`      | SQL Injection — auth bypass, UNION, error-based |
| `xxe`               | —                  | XML External Entity — file read, SSRF via XML   |
| `command-injection` | `cmd`, `command`   | OS Command Injection — shell execution          |
| `open-redirect`     | `redirect`         | Open Redirect — URL redirect to attacker domain |
| `path-traversal`    | `traversal`, `lfi` | Path Traversal — directory traversal            |

Use either the full name or any short alias:

```bash
# These are equivalent:
vulcn run -p sqli
vulcn run -p sql
vulcn run -p sql-injection
```

## Custom Payload Files

Create your own payload files for domain-specific testing:

```yaml
# custom-payloads.yml
version: "1"
payloads:
  - name: my-xss-payloads
    category: xss
    description: Custom XSS for our app
    payloads:
      - "<script>alert('custom')</script>"
      - "<img src=x onerror=alert('custom')>"
    detectPatterns:
      - "alert\\('custom'\\)"

  - name: my-sqli-payloads
    category: sqli
    description: Database-specific SQLi
    payloads:
      - "'; EXEC xp_cmdshell('whoami')--"
      - "1; SELECT pg_sleep(5)--"
```

Load in config:

```yaml
plugins:
  - name: "@vulcn/plugin-payloads"
    config:
      files:
        - ./custom-payloads.yml
```

Or via CLI:

```bash
vulcn run -f ./custom-payloads.yml
```

## Programmatic Usage

```typescript
import {
  loadPayloadBox,
  resolvePayloadType,
  getPayloadBoxTypes,
  isValidPayloadName,
  loadFromFile,
} from "@vulcn/plugin-payloads";

// Fetch payloads from PayloadsAllTheThings
const xssPayloads = await loadPayloadBox("xss", 50);
const sqliPayloads = await loadPayloadBox("sqli", 50);

// Resolve short aliases
resolvePayloadType("sqli"); // → "sql-injection"
resolvePayloadType("cmd"); // → "command-injection"
resolvePayloadType("lfi"); // → "path-traversal"

// Check if a name is valid
isValidPayloadName("sqli"); // → true
isValidPayloadName("foo"); // → false

// Load from file
const customPayloads = await loadFromFile("./my-payloads.yml");

// List available types
const types = getPayloadBoxTypes();
// ["xss", "sql-injection", "xxe", "command-injection", "open-redirect", "path-traversal"]
```
