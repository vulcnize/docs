## Prerequisites

<Info>**Requirements:** - Node.js v18 or higher - npm, pnpm, or yarn</Info>

## Installation

<Steps>
<Step title="Install the CLI">

<CodeGroup>

```bash npm
npm install -g vulcn
```

```bash pnpm
pnpm add -g vulcn
```

```bash yarn
yarn global add vulcn
```

</CodeGroup>

</Step>

<Step title="Initialize configuration">

Create a `vulcn.config.yml` file with default plugins:

```bash
vulcn init
```

This creates a config file with the official detection plugins pre-configured:

```yaml
# vulcn.config.yml
version: "1"
plugins:
  - name: "@vulcn/plugin-payloads"
  - name: "@vulcn/plugin-detect-xss"
    config:
      detectDialogs: true
      detectConsole: true
      severity: high
settings:
  browser: chromium
  headless: true
```

</Step>

<Step title="Install browsers (optional)">

Vulcn uses Playwright for browser automation. Browsers are installed automatically on first use, but you can install them manually:

```bash
vulcn install
```

Or install all browsers:

```bash
vulcn install --all
```

</Step>
</Steps>

## Record Your First Session

<Steps>
<Step title="Start recording">

Open a browser and record your interactions with the target application:

```bash
vulcn record https://example.com --output session.vulcn.yml
```

<Tip>
  The browser opens and records everything you do. Fill out forms, click
  buttons, navigate pagesâ€”Vulcn captures it all.
</Tip>

</Step>

<Step title="Interact with the app">

In the browser:

1. Navigate to a page with input fields
2. Fill out a form (login, search, comment, etc.)
3. Submit the form
4. Close the browser when done

</Step>

<Step title="Session saved">

Your interactions are saved to `session.vulcn.yml`:

```yaml
name: Session 2026-02-06
startUrl: https://example.com
browser: chromium
steps:
  - id: step_001
    type: navigate
    url: https://example.com
    timestamp: 1707192000000
  - id: step_002
    type: fill
    selector: input[name="search"]
    value: test query
    timestamp: 1707192005000
  - id: step_003
    type: click
    selector: button[type="submit"]
    timestamp: 1707192010000
```

</Step>
</Steps>

## Or, Auto-Crawl Instead

Skip manual recording entirely â€” let Vulcn discover forms automatically:

```bash
vulcn crawl https://example.com
```

This finds all forms and injectable inputs across the site and generates session files for each one. See [vulcn crawl](/cli/crawl) for details.

## Run Security Tests

<Steps>
<Step title="Execute tests">

Run the recorded session with security payloads:

```bash
vulcn run session.vulcn.yml
```

By default, Vulcn fetches `xss` payloads from PayloadsAllTheThings and uses the `@vulcn/plugin-detect-xss` plugin for detection.

</Step>

<Step title="Choose payload types">

Test with specific payload types:

```bash
vulcn run session.vulcn.yml -p xss sqli
```

Available types: `xss`, `sqli`, `xxe`, `cmd`, `redirect`, `traversal`

All payloads are fetched from [PayloadsAllTheThings](https://github.com/swisskyrepo/PayloadsAllTheThings).

</Step>

<Step title="Review findings">

If vulnerabilities are found, you'll see output like:

<Step title="Generate a report">

Generate a professional HTML security report:

```bash
vulcn run session.vulcn.yml --report html
```

This creates a `vulcn-report.html` file and opens it in your browser. The report includes a risk gauge, severity breakdown, and detailed finding cards with payloads and evidence.

For CI/CD pipelines, use JSON:

```bash
vulcn run session.vulcn.yml --report json --report-output ./reports
```

</Step>
</Steps>
