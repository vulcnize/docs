## What is Vulcn?

Vulcn is a **modern, fast penetration testing framework** designed for developers and security engineers who want to find vulnerabilities without the complexity of legacy tools. Instead of manually testing each input, Vulcn lets you:

1. **Record** interactions once (browser clicks, form submissions, navigation)
2. **Crawl** to auto-discover forms with optional authenticated crawling
3. **Replay** sessions with tiered scanning â€” Tier 1 HTTP fast scan + Tier 2 browser replay
4. **Detect** real vulnerabilities via execution-based detection (not just pattern matching)
5. **Report** findings with branded HTML dashboards, SARIF for GitHub Code Scanning, JSON for CI/CD, or YAML

## Architecture

Vulcn v0.7.0 uses a modular architecture with **drivers** for different targets and **plugins** for detection and reporting:

## Key Features

  "@vulcn/plugin-detect-reflection" - name: "@vulcn/plugin-report" ```
</Accordion>

<Accordion title="ðŸŽ¯ Execution-Based Detection" icon="bullseye">
  Unlike pattern-matching tools, Vulcn detects **actual JavaScript
  execution**â€”when `alert()` fires, you know the XSS is real. No false positives

</Accordion>

  <Accordion title="ðŸ“Š Security Reports" icon="chart-bar">

## How It Works

<Steps>
  <Step title="Record or Crawl">
    Record browser interactions manually, or use `vulcn crawl` to auto-discover
    forms. For authenticated apps, `vulcn store` encrypts credentials and
    `--creds` enables login-then-crawl.
  </Step>
  <Step title="Choose Payloads">
    Select from community-curated payload types (xss, sqli, xxe, cmd, redirect,
    traversal), or load custom ones from files.
  </Step>
  <Step title="Run Tests (Tiered)">
    **Tier 1:** HTTP fast scan replays requests via `fetch()` at ~50ms/payload,
    detecting reflected content. **Tier 2:** Browser replay confirms findings
    with real JavaScript execution monitoring.
  </Step>
  <Step title="Review Reports">
    Get detailed HTML dashboards, SARIF for GitHub Code Scanning, JSON for CI/CD
    pipelines, or YAML reportsâ€”with risk scores, severity breakdowns, and
    evidence for every finding.
  </Step>
</Steps>

## Quick Example

```bash
# Initialize configuration
vulcn init

# Record a session (opens browser)
vulcn record https://vulnerable-app.com --output session.vulcn.yml

# Run security tests
vulcn run session.vulcn.yml

# Run with multiple payload types and generate a report
vulcn run session.vulcn.yml -p xss sqli -r html

# List available payload types
vulcn payloads
```
