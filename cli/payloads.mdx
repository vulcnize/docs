## Usage

```bash
vulcn payloads [options]
```

## Options

| Option                      | Description                         |
| --------------------------- | ----------------------------------- |
| `-c, --category <category>` | Filter by category                  |
| `-f, --file <file>`         | Also show payloads from custom file |

## Description

The `payloads` command lists all available payload types. Payloads are fetched on demand from [PayloadsAllTheThings](https://github.com/swisskyrepo/PayloadsAllTheThings), the largest community-curated collection of security payloads.

## Custom Payloads

For security experts who need domain-specific payloads, create a custom payload file:

```yaml
# my-payloads.yml
version: "1"
payloads:
  - name: my-xss-payloads
    category: xss
    description: Custom XSS payloads for my application
    payloads:
      - "<script>alert(document.domain)</script>"
      - "<img src=x onerror=alert(1)>"
      - "javascript:alert(1)"
    detectPatterns:
      - "alert\\("

  - name: my-sqli-payloads
    category: sqli
    description: Custom SQLi payloads
    payloads:
      - "' OR '1'='1"
      - "1; DROP TABLE users--"
      - "1 UNION SELECT * FROM users"
```

Use with `--payload-file`:

```bash
vulcn run session.vulcn.yml -f ./my-payloads.yml
```

Or configure in `vulcn.config.yml`:

```yaml
plugins:
  - name: "@vulcn/plugin-payloads"
    config:
      files:
        - ./my-payloads.yml
```
